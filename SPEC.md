### 1.2 모바일 레이아웃

- 프리셋: 가로 스크롤 칩
- 결과 박스: 풀폭 + 아래에 [복사], [새로 만들기] 버튼 2개 가로 배치
- 옵션: 세로 스택
- 길이 슬라이더 옆에 숫자 입력 박스(미세 조정용)

### 1.3 페이지 진입 시 기본 동작

- "일반" 프리셋이 선택된 상태로 시작
- 페이지 로드 즉시 한 번 비밀번호 자동 생성 (사용자가 버튼을 누르지 않아도
  바로 복사 가능하도록)
- 옵션이 바뀌면 즉시 자동 재생성 (UI 지연 없게)

## 2. 옵션별 동작 정의

### 2.1 길이 (length)

- **범위**: 4 ~ 64
- **기본값**: 12
- **UI**: 슬라이더 + 숫자 입력 (양방향 동기화)
- **검증**:
  - 4 미만 → 4로 보정
  - 64 초과 → 64로 보정
  - 선택된 카테고리 수보다 길이가 작으면 → 길이를 카테고리 수에 맞춰 자동 증가
    (예: 4개 카테고리 선택 + 길이 3 → 길이 4로 보정 + 토스트로 안내)

### 2.2 문자 종류 (4종)

| 키 | 라벨 | 문자 풀 |
|---|---|---|
| `uppercase` | 대문자 | `A-Z` |
| `lowercase` | 소문자 | `a-z` |
| `digits` | 숫자 | `0-9` |
| `symbols` | 특수문자 | (아래 모드별) |

- **최소 1개 선택 필수**. 마지막 1개 해제하려고 하면 거부 (스위치 원위치).
- **체크된 카테고리 각각에서 최소 1자 보장**. 나머지는 합쳐진 풀에서 추출 후
  전체 셔플. 결과의 순서 편향이 없어야 함.

### 2.3 특수문자 모드 (symbols 체크 시에만 활성)

| 모드 | 문자 풀 | 설명 |
|---|---|---|
| 전체 (`full`) | `` !@#$%^&*()_+-=[]{}|;:,.<>?/~`'" \ `` | 거의 모든 일반 특수문자 |
| 일부 (`safe`) | `! @ # $ % ^ & *` | 호환성·보안 양쪽을 모두 만족하는 8종 |

- 기본값: `safe`.
- **"일부" 풀 선정 기준 (2026-05-19 확정)**:
  1. **호환성** — 대부분의 일반 웹사이트가 비밀번호로 허용하는 문자만 포함.
  2. **보안** — 서버측 입력 검증·이스케이프 실수로 사고가 잦은 문자는 제외.
     SQL injection / XSS / command injection / path traversal 등의 공격
     벡터로 빈번히 쓰이는 다음 문자는 "일부" 풀에서 제외하고 "전체"에만
     포함한다: `.` `'` `"` `;` `<` `>` `\` `/` `` ` `` `|` `(` `)` `?` `,` `:`
- "제외"는 별도 모드를 두지 않고 **`symbols` 토글 자체를 끄는 것**으로 표현
  (UI 단순화). 내부 타입에는 `off` 가 남아있어 토글이 꺼지면 동치 동작.
- 모드 버튼 아래 현재 풀의 전체 문자를 모노스페이스 + 공백 구분으로 표시
  (`! @ # $ % ^ & *`) — 사용자가 어떤 문자가 들어가는지 즉시 확인 가능.

### 2.4 혼동되는 문자 제외 (`excludeConfusable`)

- 기본값: **켜짐 (true)**
- 활성 시 다음 문자를 모든 풀에서 제거:
  - 영문자: `O`, `o`, `I`, `l`
  - 숫자: `0`, `1`
- 이유: 종이에 적거나 폰트에 따라 헷갈리기 쉬움. 1회용이라 더더욱 입력 실수
  최소화가 중요.
- UI 라벨의 예시(`O/0`, `l/1`, `I`)는 가독을 위해 **모노스페이스(고정폭) 글꼴**
  로 표시.

### 2.6 첫 글자 영문자 보장

- 생성된 비밀번호의 **첫 글자는 항상 영문자(대 또는 소)**.
- 이유: 일부 사이트가 "비밀번호는 영문자로 시작" 정책을 강제하며, 그 외
  사이트에서도 영문자 시작이 호환성 최상.
- 예외: 사용자가 **대문자와 소문자 카테고리를 모두 끈 경우** (숫자만 또는
  특수문자만으로 생성하려는 의도) 이 규칙은 적용하지 않음.
- 구현: generator 마지막 셔플 후 첫 글자가 영문자가 아니면 첫 영문자 위치와
  swap. 카테고리별 최소 1자 보장 및 균등 분포에는 영향 없음.

### 2.5 자동 재생성 트리거

다음 조건 중 하나라도 발생하면 즉시 비밀번호 재생성:
- 옵션 어느 하나라도 변경
- 프리셋 변경
- [새로 만들기] 버튼 클릭

생성된 비밀번호는 **state에만** 존재. 새로고침/탭 닫기 시 소실.

## 3. 위험 패턴 감지 (Danger Patterns)

생성된 비밀번호에 다음 패턴이 검출되면 빨간 경고 배너 + [새로 만들기]를 강조
표시. 자동 재생성은 하지 않음 (사용자 의사를 존중, 단 1초 안에 본인이 누르도록
유도).

### 3.1 카테고리별 패턴 사전

#### A. 키보드 시퀀스 (대소문자 무관)
- `qwerty`, `qwertz`, `azerty`, `asdf`, `zxcv`, `qaz`, `wsx`, `edc`
- `qwer`, `asdfgh`, `zxcvbn`, `1q2w3e`, `1qaz2wsx`

#### B. 숫자 시퀀스
- `0123`, `1234`, `2345`, … (연속 4자리 이상)
- `9876`, `8765`, … (역방향 연속 4자리 이상)
- `1111`, `0000` 등 같은 숫자 4자 이상 반복

#### C. 흔한 비밀번호 어휘 (대소문자 무관, leet 변형 포함)
- `password`, `passwd`, `pass`, `passw0rd`, `p@ssword`
- `admin`, `administrator`, `root`, `user`, `guest`, `test`
- `login`, `welcome`, `letmein`, `monkey`, `dragon`, `master`
- `iloveyou`, `princess`, `superman`, `batman`
- `사랑해`, `안녕`, `비밀번호` (한글 흔한 변형 — 추후 확장)

#### D. 흔한 날짜 패턴
- 4자리 연속 숫자 중 `19xx`, `20xx` (연도로 해석되는 경우) — 단독 출현 시 경고
- `MMDD`, `DDMM` 형태가 단독 토큰처럼 보일 때 (휴리스틱, false positive 허용)

#### E. 같은 문자 4회 이상 반복
- `aaaa`, `1111`, `!!!!` 등

### 3.2 매칭 규칙

- **부분 문자열 매칭**(case-insensitive 기본). leet 변환 후에도 매칭:
  - `0→o`, `1→i/l`, `3→e`, `4→a`, `5→s`, `7→t`, `@→a`, `$→s`
- 매칭된 패턴은 경고 메시지에 함께 표시 ("`qwerty` 패턴 포함")
- 둘 이상 매칭 시 가장 위험한 1~2개만 표시 (UI 과밀 방지)

### 3.3 강도 표시와의 관계

- zxcvbn 점수가 3 이상이어도 위험 패턴이 잡히면 경고 표시 (정직성 우선)
- 반대로 zxcvbn이 낮아도 위험 패턴이 없으면 그냥 "약함" 만 표시

## 4. 프리셋

프리셋 클릭 시 옵션 전체가 그 값으로 교체됨. 이후 사용자가 옵션을 수정하면
프리셋 선택은 "사용자 지정" 상태로 전환됨 (탭 강조 해제).

### 4.1 프리셋 표

| 프리셋 | 길이 | 대문자 | 소문자 | 숫자 | 특수 | 특수 모드 | 혼동 제외 | 비고 |
|---|---|---|---|---|---|---|---|---|
| 일반 | 12 | ✓ | ✓ | ✓ | ✓ | safe | ✓ | 가장 호환성 높은 기본값 |
| 강화 | 20 | ✓ | ✓ | ✓ | ✓ | full | ✓ | 길이 + 풍부한 특수문자 |
| 네이버 | 10 | ✓ | ✓ | ✓ | ✓ | safe | ✓ | 8~16자 / 영·숫·특 3종 이상 |
| 구글 | 14 | ✓ | ✓ | ✓ | ✓ | full | ✓ | 8자 이상 권장, 여유 길이 |
| 은행권 | 10 | ✓ | ✓ | ✓ | ✓ | safe | ✓ | 보통 10자 + 3종 조합 의무 |
| 게임 | 12 | ✓ | ✓ | ✓ | ✗ | off | ✓ | 특수문자 미허용 사이트 다수 |

> 프리셋의 실제 사이트 정책은 사이트별로 자주 바뀝니다. 위 값은 "흔한
> 정책에 대체로 맞는" 보수적 기본값일 뿐이며, 실패 시 사용자가 직접
> 옵션을 미세 조정합니다.

### 4.2 프리셋 정의 위치

`lib/presets.ts` 에 객체 배열로 정의. 각 프리셋은 `{ id, label,
description, options: PasswordOptions }` 형태.

## 5. 사용자 시나리오

### 시나리오 1 — "이벤트 응모 사이트 가입"
1. 동료가 "이 사이트 가입하면 기프티콘 준대" 라고 링크를 보냄.
2. 회원가입 페이지에 들어가니 비밀번호 정책이 "8자 이상, 영문/숫자/특수문자
   포함" 이라고 적혀 있음.
3. 막번호 페이지를 새 탭에서 엶 → "일반" 프리셋이 이미 선택되어 있고
   `xR3$kZ9pqM2!` 같은 게 떠 있음.
4. [복사] 클릭 → 가입 폼에 붙여넣기.
5. 가입 완료. 막번호 탭은 닫음 → 비밀번호는 사라짐. 어차피 다시 안 갈 사이트.

### 시나리오 2 — "네이버 비밀번호 정책에 막혔다"
1. 다른 도구로 만든 비밀번호로 네이버 가입을 시도했는데 거절됨.
2. 막번호의 "네이버" 프리셋 클릭 → 옵션이 네이버 정책에 맞게 자동 세팅.
3. 즉석에서 새 비밀번호가 생성됨.
4. [복사] → 가입 성공.

### 시나리오 3 — "위험 패턴 경고로 재생성"
1. "길이 8, 소문자만, 특수문자 제외" 같은 약한 옵션으로 생성.
2. 결과가 `qwertabc` → 위험 패턴 `qwert` 감지, 빨간 경고 배너 표시.
3. 사용자가 [새로 만들기] 클릭 → 새로 생성된 비밀번호는 패턴 없음.
4. 강도 게이지는 여전히 "보통" 이지만 경고가 사라졌으므로 사용자는 안심하고
   복사.

## 6. 비밀번호 직접 편집 (Stage 2, 2026-05-20)

생성된 비번을 사용자가 직접 수정할 수 있다. 편집 중에는 강도·위험 패턴·
문자 구성 통계가 실시간 갱신된다.

### 6.1 진입·종료

- **진입**: 비번 박스 우측 상단 `✏️` 아이콘 버튼.
- **종료**:
  - **✓ 완료** — 편집된 문자열을 새 비번으로 확정. 프리셋 강조는 해제됨
    (사용자 지정 상태). 길이가 4 미만이면 비활성화.
  - **↩ 되돌리기** — 편집을 폐기하고 진입 시점 비번으로 복귀.
  - **Enter 키** — `✓ 완료`와 동치(길이 충족 시).

### 6.2 편집 모드 UI

- 비번 박스가 `<input>` 으로 전환. 한 줄 입력(`type="text"`,
  `inputMode="text"`, `lang="en"`, `spellCheck=false`, `autoCorrect=off`,
  `autoCapitalize=off`).
- **입력 가능 문자 = ASCII 인쇄 가능 범위(0x20–0x7E)만**. 한글·이모지·
  제어문자 등은 `onChange` 단계에서 정규식 `/[^\x20-\x7E]/g` 로 즉시 제거.
  generator 가 만들 수 있는 문자 풀과 일치하며, 한글 비번을 거부하는 대부분의
  한국 사이트와의 호환성도 보장. 사용자에게는 "한/영 전환 안 했네"가 즉시
  인지된다.
- 박스 보더가 살짝 강조됨(`border-foreground/40 + shadow-md`).
- 박스 아래 행에 실시간 통계:
  - `12/64` 길이 카운터 (코드포인트 단위)
  - 길이 < 4 이면 빨간 글씨로 `사용 불가 (최소 4자)` 라벨 + `✓ 완료` 비활성화
  - 문자 구성 `대 N · 소 N · 숫 N · 특 N` — 0 인 항목은 흐리게.
    ASCII 범위 밖 문자(한글·이모지 등)가 있으면 `기타 N` 추가.
- **옵션 영역과 프리셋은 비활성화**(`<fieldset disabled>`) — 편집 중
  옵션 변경으로 인한 자동 재생성으로 편집 손실 방지.

### 6.3 실시간 평가 디바운스

매 키스트로크마다 평가를 다시 돌리면 비용이 크므로 항목별 디바운스 적용.

| 항목 | 갱신 |
|---|---|
| 길이 카운터 | 즉시 |
| 문자 구성 (대·소·숫·특) | 즉시 |
| 강도 바 + 예상 크랙 시간 | 150ms |
| 위험 패턴 배너 | 200ms |

### 6.4 평가 인프라 (`lib/evaluator.ts`)

- 임의 문자열을 받아 평가. `generator` 와 의존성 없음 — 향후 다른 비밀번호
  생성 모드(예: 한영 매핑)에서도 재사용 가능.
- `getComposition(pwd)` 동기, `getDangerMatches(pwd)` 동기,
  `getStrength(pwd)` 비동기.
- 내부적으로 `dangerPatterns` / `strength` 모듈에 위임.

### 6.5 위험 패턴 배너의 "새로 만들기" 동작

편집 중에 사용자가 빨간 배너의 `새로 만들기` 를 누르면 **편집이 폐기되고**
현재 옵션으로 새 비번이 생성된다(편집 의도 무효화). 무심코 누르는 것을
방지하려면 향후 확인 다이얼로그를 추가할 수 있다.

## 7. 비기능 요구사항

- **성능**: 옵션 변경 → 비밀번호 재생성 → 강도 계산까지 50ms 이내 (체감 즉시).
  zxcvbn은 무거우므로 디바운스(예: 100ms) 적용 검토.
- **접근성**: 모든 컨트롤 키보드 조작 가능, `aria-label` 명시, 명도 대비 AA
  이상.
- **다크 모드**: 1차 버전에서 미지원. 추후 추가.
- **국제화**: 1차는 한국어 단일. UI 문자열은 향후 i18n 가능한 구조로
  (객체로 분리).
- **오프라인**: 100% 클라이언트라 첫 로드 후 오프라인 동작. PWA화는 추후
  검토.
